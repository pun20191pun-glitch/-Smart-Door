🖥️ คู่มือการเซตระบบตรวจจับใบหน้าและควบคุมประตู (PC Central Control)
1. การเตรียมเครื่อง PC และซอฟต์แวร์ (Prerequisites)
ก่อนเริ่มใช้งาน ต้องติดตั้งเครื่องมือลงใน PC ให้ครบถ้วน:

VS Code: สำหรับรันโค้ด Python และจัดการไฟล์

Arduino IDE: สำหรับ Flash โค้ดลงบอร์ด UNO และ ESP32

Python 3.x: พร้อมติดตั้ง Library จากโฟลเดอร์ "สิ่งที่ต้องติดตั้งลงในเครื่องก่อนใช้งาน" (สำคัญมาก: หากลงไม่ครบ ไฟล์ .py จะรันไม่ได้)

MQTT Broker & Dashboard: ใช้ PC เป็นตัวกลาง (Broker) และดูสถานะผ่านหน้าจอ Terminal หรือ Dashboard บน PC

2. การตั้งค่าบอร์ด (Hardware Flash)
เชื่อมต่อบอร์ดเข้ากับ PC และอัดโค้ดผ่าน Arduino IDE:

บอร์ด UNO: เปิดไฟล์ UNO ประตู > เลือก Port ให้ตรง > กด Upload

บอร์ด ESP32: เปิดไฟล์ ESP32 ประตู > แก้ไขข้อมูล WiFi และ IP:

ssid = "ชื่อ WiFi ของคุณ"

password = "รหัสผ่าน WiFi ของคุณ"

mqtt_server = "IP เครื่อง PC ของคุณ" (เช็คได้จากคำสั่ง ipconfig ใน CMD)

3. การจัดการข้อมูลใบหน้า (Face Database)
ทำบน PC ผ่านไฟล์ในโฟลเดอร์ DetectFaceApiProd:

การเพิ่มใบหน้า (Face Enrollment):

นำรูปใบหน้าที่ต้องการเพิ่มไปใส่ในโฟลเดอร์ facedata

รันไฟล์ trainface2.py เพื่อประมวลผลให้ระบบจดจำใบหน้าใหม่

4. ขั้นตอนการเริ่มระบบ (System Startup)
ทำตามลำดับนี้เพื่อให้ระบบเชื่อมต่อกันได้สมบูรณ์:

ตรวจสอบ Network: ให้แน่ใจว่า PC, ESP32 และ กล้อง ต่อ WiFi วงเดียวกัน

เช็ค IP & RTSP: เปิดไฟล์ facedetace4.py ใน VS Code:

เช็ค broker = "IP เครื่อง PC" ให้ตรงกับที่แก้ใน ESP32

เช็ค rtsp_url = "rtsp://..." (ใส่ IP และ User/Pass ของกล้องให้ถูกต้อง)

รันระบบ: กด Run ไฟล์ facedetace4.py

หน้าจอจะแสดงภาพจากกล้อง และเริ่มตรวจจับใบหน้า

เมื่อเจอใบหน้าที่ "ผ่านได้" ระบบจะส่งคำสั่ง MQTT ไปยัง ESP32 เพื่อสั่ง UNO เปิดประตูทันที

💡 ข้อควรระวังและการแก้ไข
IP เครื่อง PC: หากเปลี่ยนที่ทำงานหรือรีสตาร์ท Router ค่า IP ของ PC อาจเปลี่ยน (ต้องตามแก้ใน facedetace4.py และ ESP32 ประตู ให้ตรงกันเสมอ)

RTSP กล้อง: กล้องแต่ละยี่ห้อมีรูปแบบ URL ไม่เหมือนกัน ให้ตรวจสอบจากคู่มือกล้องรุ่นนั้นๆ

Terminal: สังเกตสถานะในหน้า Terminal ของ VS Code หากขึ้น "Connected to MQTT" แสดงว่าการเชื่อมต่อกับบอร์ดสำเร็จแล้ว
